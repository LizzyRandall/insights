(static) drag

Contains functions to be used with InteractJS for UI dragging and dropping. Each function takes a mouse event as the only argument.

Properties:
Name Type Description
drag.basic function

Allows element to be dragged and dropped.
drag.column function

Only allows dragging if the column object is defined and has a code.
drag.withScroll function

Same as drag.basic but compensates for dragging in a scrollable window.
drag.snapBack function

End function which sends the element back to its original location.
drag.enter function

Adds dropTarget class to the element when entering a valid drop zone.
drag.leave function

Removes the dropTarget class when leaving the drop zone.
drag.dropDisable function

Removes the dropTarget class when deactivating the drop zone.

Example
            // Basic movement
interact(element[0]).draggable({
	onmove: insights.drag.basic,
	onend: insights.drag.snapBack

// Dropping onto another element
interact(element[0]).dropzone({
	accept: '.dragColumn', // Only accept elements matching this CSS selector

	ondragenter: insights.drag.enter,
	ondragleave: insights.drag.leave,
	ondrop: dropFunction, // Custom function to execute on element drop
	ondropdeactivate: insights.drag.dropDisable
});

(static) printDB(dbElem, filename, type)

Converts a complete dashboard to a canvas element and then exports to either PNG or PDF format. Works by walking through the DOM tree of the element and converts HTML using html2canvas and SVG using FabricJS. Leaflet maps are rendered similarly but require the container to have the class print-as-map attached. Note that map exporting is unreliable, mainly due to the dynamic fetching of the tile layer. The class do-not-print can be added to any element on a plugin to prevent it from being exported. CSS attributes must be applied to SVG elements as attributes for the export to work, and so functions applySVGCSS and revertSVGCSS are provided. Finally, the canvases for each visualisation are stitched together using the visualisation X and Y coordinates and are are exported using FileSaverJS.

Parameters:
Name Type Description
dbElem DOM

HTML element container for the dashboard that needs printing.
filename string

Name of the file to save to.
type string

Type of file to export, accepts either png or pdf.

(static) printHTML(elem, filename, type)

Function to convert an HTML/SVG element to a canvas and export as a PNG or PDF. Works by walking through the DOM tree of the element and converts HTML using html2canvas and SVG using FabricJS. Leaflet maps are rendered similarly but require the container to have the class print-as-map attached. Note that map exporting is unreliable, mainly due to the dynamic fetching of the tile layer. Additionally, the Leaflet map object should be attached to this element as a jQuery data attachment, with the property mapObject. E.g. $(mapContainer).data({ 'mapObject' : map }) The class do-not-print can be added to any element on a plugin to prevent it from being exported. CSS attributes must be applied to SVG elements as attributes for the export to work, and so functions applySVGCSS and revertSVGCSS are provided. Finally, the canvas is exported using FileSaverJS.

(static) urlFilters(dbObj)

Parse URL query string and apply filters to dashboard if applicable. Accepts n filters by specifying the column ID (e.g. Products.Type) and the value (; separate for multiple). The query string arguments should be of the form filter1, filter2, ..., op1, op2, ... and val1, val2, .... If matching dashboard prompts are found, the default values are updated. All visualisations are checked, but filters are only applied if they contain either a column in the criteria with a matching column ID or an unprotected filter in the query with a matching column ID.

Name Type Description
